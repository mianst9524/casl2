# CASL2 学習シミュレーター

CASL2 で書いたプログラムをアセンブルし、COMET II 仮想計算機上での実行を可視化する学習用シミュレーター。主目的は試験対策ではなく、低レイヤの仕組み（ノイマン型アーキテクチャ＝命令もデータも同じメモリ語、および C 言語との対応）を学習者に「腹落ちさせる」こと。JIS X 0205 の範囲とは互換を保つ。

## Language

**CASL2**:
COMET II 用のアセンブリ**言語**。`LD GR1,GR2` のようなソースを記述する対象。JIS X 0205 で規定。
_Avoid_: 「機械語」（CASL2 はアセンブリ言語であって機械語そのものではない）

**COMET II**:
CASL2 が動作する仮想**計算機**。16 ビット語、汎用レジスタ GR0〜GR7、スタックポインタ・プログラムレジスタ・フラグレジスタ、65536 語の主記憶を持つ。シミュレーターが実行を再現する対象はこの機械。
_Avoid_: 「CPU」（文脈上は COMET II と呼ぶ）

**シミュレーター**:
本プロジェクトの成果物そのもの。CASL2 ソースのアセンブル、COMET II の状態（レジスタ・メモリ）、実行の歩みを可視化するツール。
_Avoid_: 「エミュレーター」（実機を持たない仮想計算機なので「シミュレーター」で統一）

**語（word）**:
COMET II の記憶・演算の基本単位。16 ビット。主記憶もレジスタもすべて 1 語単位で扱う。命令もデータもこの 1 語の集まりにすぎない。
_Avoid_: 「バイト」（COMET II にバイト単位のアクセスは無い）

**主記憶（メモリ）**:
0〜65535 番地の**フラットな語配列**。ハードウェアが認識する区画は存在しない。全番地が同一の 16 ビット語で、そこに置かれた語が命令かデータかは「どう使われるか」だけで決まる。
_Avoid_: 「セグメント」「コードセグメント／データセグメント」（COMET II にハードのセグメント分割は無い）

**スタック**:
SP（スタックポインタ）が指す場所として動的に定まる、唯一ハードが扱う「領域」。`PUSH`/`CALL` で高位番地から**下方向**へ伸び、`POP`/`RET` で戻る。固定の番地範囲を持つ静的な区画ではない。

**領域ラベル（役割の色分け）**:
メモリビュー上で各語に重ねる**人間向けの注釈**（命令 / データ / スタック など）。COMET II のハードが持つ区画ではなく、あくまで表示上の色分け。同じ語でも、PR が指して実行されれば命令、命令のオペランドや LD/ST で参照されればデータ。「命令とデータは同じメモリ語」という本プロジェクトの核心を支える概念。

**ヒープ**:
本シミュレーターでは**用いない**。COMET II／CASL2 に動的メモリ確保（`malloc` 相当）の機構が無いため。C 言語対応の文脈で「C にはヒープがあるが COMET II には無い」という対比を示す教材的価値はあるが、メモリ領域としては設けない。
_Avoid_: メモリ領域名として「ヒープ」を使うこと。

## 命令の3分類

CASL2 のソース行は3種類あり、「アセンブル結果としてメモリに何が置かれるか」が異なる。これは「命令＝データ」を腹落ちさせる前提となる区別。

**機械命令**:
COMET II が実行する命令。アセンブルすると機械語の語になり、メモリに置かれ、PR に指されたとき実行される。`LD`/`ST`/`LAD`/`ADDA`/`CPA`/`JUMP`/`CALL`/`RET`/`PUSH`/`POP`/`SVC` など。
_Avoid_: 「命令」と言うとき、疑似命令やマクロ命令まで含めて曖昧に使うこと。

**疑似命令（アセンブラ命令）**:
アセンブル時にアセンブラが処理する指示で、COMET II は実行しない。`START`/`END`（プログラムの入口・終端を示すのみで、メモリに語を生まない）、`DS`（語領域を確保する）、`DC`（定数の語をメモリに置く）。`DC` が置いた語は「データ」として参照されるのが普通だが、実体はメモリ語なので命令として実行されることもありうる（PR 侵入デモ）。

**マクロ命令**:
1 行が複数の機械命令（や `SVC`）へ展開される命令。`IN`/`OUT`/`RPUSH`/`RPOP`。展開後の機械命令語がメモリに並ぶ。

**ラベル**:
番地に付ける名前。アセンブル時に番地（数値）へ解決され、実行時のメモリには番地しか残らない（名前は存在しない）。

## 割り込み

「割り込み」は2種類あり、必ず区別する。

**ソフトウェア割り込み（SVC）**:
プログラムが `SVC` 命令で**自ら**スーパバイザ（OS）へ制御を渡す同期的な仕組み。JIS X 0205 の**標準範囲内**。スーパバイザ呼出しとも呼ぶ。
_Avoid_: これを単に「割り込み」と呼ぶこと（ハードウェア割り込みと紛らわしい）。

**ハードウェア割り込み**:
タイマーやキー入力などの**外部イベント**が、実行中の命令の切れ目で処理を**非同期に**中断し、状態を退避してハンドラへ飛び、復帰する仕組み。**COMET II 標準には存在せず、本シミュレーター独自の教材用拡張**。UI 上で「拡張」と明示する。
_Avoid_: 標準機能であるかのように見せること。

**割り込みハンドラ**:
ハードウェア割り込み発生時に実行される処理ルーチン。その先頭番地（割り込みベクタ）へ PR が移る。

**割り込み許可フラグ（IE）**:
ハードウェア割り込みを受け付けるか否かを表す拡張状態。ハンドラ実行中は受け付けないなどの制御に使う。FR の3フラグ（OF/SF/ZF）とは別物。

**RETI（割り込み復帰）**:
ハードウェア割り込みハンドラから復帰する**拡張命令**。スタックから PR と FR を復帰し、IE を 1 に戻す。標準命令 `RET` が PR のみ復帰するのと対比され、「割り込みは非同期ゆえフラグまで退避・復帰が要る」ことを示す。
_Avoid_: `RET` と混同すること（`RET` は標準・PR のみ復帰、`RETI` は拡張・PR＋FR＋IE 復帰）。

## 実行・可視化

**ステップ**:
1 命令のフェッチ・デコード・実行の 1 サイクル。本シミュレーターの実行単位。
_Avoid_: 学習単位の「レッスン」と混同すること（ステップは実行の 1 サイクル、レッスンは学習サンプルの 1 段）。

**ステップ差分（実行履歴）**:
1 ステップごとに記録する「変化したレジスタ・フラグ・メモリ語の**旧値→新値**」の記録。逆実行（ステップバック）と「変更ハイライト」の両方の土台になる単一の仕組み。
_Avoid_: 「スナップショット」（全状態の丸ごと保存ではなく、変化分だけの差分を指す）

**逆実行（ステップバック）**:
記録したステップ差分を逆順に適用し、機械の状態を 1 ステップ前へ戻す操作。自己書き換えで変わった命令語も正しく巻き戻す。

**変更ハイライト**:
直前の 1 ステップで変化した箇所（レジスタ／メモリ語）を画面上で強調する表示。ステップ差分から得る。

## 学習構成（カリキュラム）

**レッスン**:
学習用サンプルを易→難へ一直線に並べた 1 段。各レッスンは前のレッスンに新概念を 1 つだけ足す（機能の積み上げ）。1 から始まる通し番号を持ち、フェーズ1（純粋な CASL2/COMET II）→ 幕間（COMET II の本質・C に無い世界）→ フェーズ2（C の主要機能との対応）の順に進む。
_Avoid_: 「ステップ」（ステップは実行の 1 サイクル。学習単位はレッスン）。

**新概念**:
そのレッスンで初めて登場する概念。1 レッスン＝前段＋新概念 1 つ、という縛りを成立させる単位。

**回収**:
フェーズ2 の各レッスンが、フェーズ1 で純粋な CASL2 として学んだ仕組みに「これが C の `if`／`while`／関数だ」と意味を重ね直すこと。フェーズ1 の知識をフェーズ2 で意味づけし直す構造。

## 例題対話

> **学習者**: メモリの 3 番地、これは命令ですか？データですか？
>
> **講師**: それは「どう使われるか」次第です。中身は `#1021` という 16 ビットの**語**が 1 つあるだけ。**領域ラベル**で命令色に塗ってあるのは、いま PR がこの番地を指して機械命令としてデコードする予定だから。でも同じ語を `LD` で読めば、ただの数値 4129 という**データ**です。命令とデータの違いはメモリの中には無い。
>
> **学習者**: じゃあ `DC` で置いた語を、うっかり実行してしまったら？
>
> **講師**: まさにそれが PR 侵入デモ。`DC` は**疑似命令**で、置かれるのはただの語。PR がそこへ進めば COMET II は機械命令としてデコードして実行します。1 ステップ進めて、**変更ハイライト**でどのレジスタが変わったか見てみましょう。おかしくなったら**逆実行**で 1 個戻せます——**ステップ差分**を逆に当てるだけなので。
>
> **学習者**: 途中でキーが押されたら割り込みが起きますよね。`CALL` とどう違うんですか？
>
> **講師**: 大事な区別です。`CALL` はプログラムが自分で呼ぶ。**ハードウェア割り込み**は外部イベントが**非同期**に割り込む——だから FR（フラグ）まで壊れると困る。`CALL` の戻りは `RET` で PR だけ戻せばいいけれど、割り込みの戻りは `RETI`（拡張）で PR と FR、それに IE まで戻す。ちなみに `SVC` は「割り込み」と名前は似ていても、プログラムが自分で OS を呼ぶ**ソフトウェア割り込み**で、これは標準の仕組みです。
