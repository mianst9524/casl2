# 実行はメモリ上の生の語を毎ステップ解釈する（事前デコードした IR を持たない）

命令とデータが同一のメモリ語でしかないこと、自己書き換えコード、PR のデータ領域侵入を可視化することが本シミュレーターの中心目的。これを成立させるため、アセンブラは CASL2 ソースを 16 ビット機械語へ変換して**データと共有のフラットなメモリ配列**へ配置し、実行エンジンは毎ステップその配列から語をフェッチ・デコード・実行する。命令を事前に内部表現（IR/AST）へコンパイルして実行する方式は採らない。ソース行 ↔ 番地 ↔ 機械語の対応は**表示専用のメタデータ（ソースマップ）**として別に保持し、実行の真実の源にはしない。

## Considered Options

- **生メモリ毎ステップ解釈（採用）** — 真実の源はメモリ語配列ただ一つ。命令＝データ・自己書き換え・データ実行がそのまま成立する。
- **事前デコード IR 方式（不採用）** — アセンブル時に命令を内部オブジェクトへ変換して実行。高速かつ実装が単純だが、命令を不変として扱うため上記の教材的可視化が表現できない。

## Consequences

人間のステップ実行速度では IR 方式との性能差は無意味なので、教育目的を優先する。実装上は「メモリへの書き込みが次の命令フェッチに即反映される」ことを保証し、デコード結果をキャッシュしない（あるいは書き込み時に無効化する）こと。
