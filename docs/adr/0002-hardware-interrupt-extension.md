# COMET II を拡張してハードウェア割り込みを実装する（JIS 標準からの意図的逸脱）

低レイヤの仕組み（外部イベントによる非同期な制御の移動）を学習者に体験させるため、JIS X 0205 に存在しないハードウェア割り込み機構を独自拡張として実装する。標準の `SVC`（ソフトウェア割り込み）はそのまま提供し、ハードウェア割り込みは UI 上で「拡張」と明示して標準と区別する。

## モデル

- **追加状態**: 割り込み許可フラグ IE、割り込みベクタレジスタ IVR（ハンドラ先頭番地）
- **発火**: UI の「割り込み発火」ボタン（最小構成。将来タイマー等の擬似デバイスを追加可能）
- **発生時**（IE=1 で現命令の完了後）: FR と PR をスタックへ退避 → IE←0 → PR←IVR
- **復帰**: 拡張命令 `RETI` が PR と FR を復帰し IE←1
- 標準命令 `RET` は PR のみ復帰で不変

## メカニズムの確定（実装設計で詰めた内容）

当初は「IE・ベクタの設定手段」を未定としていたが、実装に向け以下に確定した。

- **拡張命令（いずれも1語、UI で標準と別色表示）**:
  - `RETI`（オペコード `#82`）— 割り込み復帰。PR と FR を pop、IE←1
  - `EI`（`#83`）— 割り込み許可（IE←1）
  - `DI`（`#84`）— 割り込み禁止（IE←0）
  - `SIV adr,x`（`#85`）— 割り込みベクタ設定（IVR ← 実効アドレス）
- **ベクタはメモリ番地ではなく専用レジスタ IVR とする**。固定番地方式はスタック（高位番地から下降）やプログラム領域と衝突するため。IVR は UI の拡張状態表示に IE と並べて出す。
- **退避フォーマット**: FR の OF/SF/ZF を 1 語へパック（IE は FR 語に含めず、退避シーケンスで明示的に IE←0、`RETI` で IE←1）。
- **多重ネスト割り込みは非対応**（退避時に IE←0 とし、ハンドラ中は再発火しない。簡潔さ優先）。
- **逆実行**: 割り込み発生（退避）は命令とは別の `interrupt-entry` ステップ差分として記録し、`stepBack` で退避を巻き戻す。発火で消費した割り込み番号も差分に持たせ、巻き戻しで保留キューへ戻す。

## Considered Options

- **標準厳守（不採用）** — JIS 範囲に限ると割り込みは扱えず、「割り込みとは何か」を体験させられない。
- **拡張として追加し明示（採用）** — 互換性を保ちつつ、拡張部分を色等で明示することで「標準/拡張」の境界も学べる。

## Consequences

`RET`（PR のみ）と `RETI`（PR＋FR＋IE）の対比により「割り込みは非同期ゆえフラグまで退避が要る」ことを示せる。拡張要素は常に標準と視覚的に区別すること。
