# 実行はステップ差分ジャーナルで記録する（全状態スナップショットを採らない）

逆実行（ステップバック）と「変更ハイライト」を成立させるため、実行エンジンは 1 ステップごとに「変化したレジスタ・フラグ・SP・PR・IE・メモリ語の**旧値→新値**」だけを差分として記録する。メモリ全体を含む全状態の丸ごと保存（スナップショット）は採らない。

ADR-0001（生メモリ毎ステップ解釈）と対になる決定で、状態を変更する経路はすべてこのジャーナル記録層を通す。

## Considered Options

- **差分ジャーナル（採用）** — 逆実行と変更ハイライトを単一の仕組みで賄える。記録は変化分のみで軽量。
- **全状態スナップショット（不採用）** — 実装は単純だが 65536 語を毎ステップ複製しメモリ消費が大きく、「この命令で何が変わったか」は別途差分計算が要る。
- **逆実行なし（不採用）** — 巻き戻しはリセット＆再実行のみ。学習体験が劣る。

## Consequences

自己書き換えで変化した命令語も差分に含まれるため正しく巻き戻せる。状態変更を直接行わずジャーナル経由にする規律が、実装全体に必要になる。
