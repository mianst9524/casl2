# 学習サンプルを1本のレッスン階段として構成する

サンプルを「機能の積み上げ」順に並べた一直線のレッスン階段（17レッスン）として構成する。各レッスンは前段に新概念を1つだけ足す。フェーズ1（純粋な CASL2/COMET II・1〜9）→ 幕間（COMET II の本質・C に無い世界・10〜12）→ フェーズ2（C の主要機能との対応・13〜17）の順に登り、最終的に C の主要機能を想像できる地点に至る。

## 決定の要点

- **軸は機能の積み上げ × 直線**。難易度タグや規模順ではなく、概念の依存関係（値→格納→計算→分岐→ループ→配列→関数→引数渡し→入出力）に沿って一直線に並べる。
- **回収はコードを共有しない**。フェーズ2 の各レッスンは、対応するフェーズ1 レッスンと同じコードを再利用せず、各々に最適なコードを書き、`recallsLesson` で概念的にリンクするだけ。
  - 理由: コード一致による「文字どおりの再会」効果より、各レッスンを文脈に最適化する自由を優先した。将来この方針を知らない読み手が「親切心でコードを揃えて直す」恐れがあるため明記する。
- **UI は2系統を維持する**。レッスン1〜12 はエディタ上部のサンプルドロップダウン（`SAMPLES`）、13〜17 は C 対応ペイン（`CCORR_EXAMPLES`、左右ハイライト専用 UI）。1本の階段だが、フェーズ2 の対訳ハイライトは専用ペインの価値が高いため統合しない。代わりに両系統に共通メタ（`lesson`/`phase`/`concept`/`recallsLesson`/`watch`）を持たせ、段階性を伝える。

## 用語

学習単位は「レッスン」と呼ぶ。実行の1サイクルである「ステップ」とは別物（CONTEXT.md 参照）。
